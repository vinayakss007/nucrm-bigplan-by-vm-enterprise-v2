# Visual-first landing page redesign and branding system

The landing page was gutted and rebuilt around a dark-first aesthetic with CSS-only animations, three external Unsplash images, glassmorphism cards, and a condensed feature/pricing layout. Supporting documentation (brand guide, copy guide, color tokens, animation library) was added under a new `branding/` folder. The previous 9-feature, 4-plan, testimonial-based page is replaced by a minimal 4-feature, 3-plan, image-driven layout targeting a "Linear/Vercel" visual style.

**Watch for:** The page hard-codes Unsplash URLs that will break when Unsplash rate-limits or removes the photos (confirmed). The dark-only design removes the light-mode experience the rest of the app still supports (confirmed). Several defined animations (`orbit`, `blob-morph`) ship in globals.css but are never used (confirmed).

## High-level view

The landing page drops all dynamic behavior (the `AnimatedCounter` with IntersectionObserver) in favor of a single `useEffect → setVisible(true)` fade-in, which means the page has no scroll-triggered interactivity. Content is drastically reduced: 9 features → 4, 4 plans → 3, testimonials removed entirely. The visual weight shifts from text to images and gradient backgrounds.

The branding folder adds 1,138 lines of documentation and CSS variables that are never imported or referenced by the landing page or any other file. They exist as a reference artifact. The landing page's inline Tailwind classes use hard-coded values (`from-violet-600`, `bg-slate-950`) rather than the CSS custom properties defined in `branding/assets/colors.css`, so the two are disconnected.

The Unsplash integration adds `images.unsplash.com` to Next.js `remotePatterns`, which is the correct mechanism for allowing external images with `next/image`. However, the three image URLs use fixed query parameters (`?w=800&h=600&fit=crop`) that lock the source resolution regardless of viewport.

<summary>Issues (5)</summary>

1. **Hard-coded Unsplash URLs** — Three images reference specific Unsplash photo IDs with no fallback. If photos are removed or rate-limited, the page shows broken images. Host the images locally or in a CDN under project control.
2. **Dark-only page vs light-mode app** — `bg-slate-950 text-slate-100` is unconditional. The nav, auth pages, and dashboard still support light mode via `dark:` prefixes. Users following "Start Free" jump from a dark page to a potentially light app, creating a jarring transition. Add `dark:` conditional classes or force `dark` class on the root.
3. **Dead animation classes** — `animate-orbit`, `animate-blob-morph`, and `animate-typing-cursor` (the `|` character as a typing cursor is static—there's nothing being "typed") ship in production CSS but serve no meaningful purpose. Remove unused keyframes or implement the typing effect properly.
4. **Branding assets disconnected from implementation** — `branding/assets/colors.css` and `branding/assets/animations.css` define 446 lines of CSS variables and keyframes that no file imports. The landing page duplicates the same animations in `globals.css` with different names. Either import the branding assets or remove the duplication.
5. **No image `alt` text quality** — Alt attributes are generic ("NuCRM Dashboard", "AI Technology", "Integration Technology"). For SEO and accessibility on a landing page, these should describe what's visually depicted, not what concept the image represents.

## External image dependency and optimization gap

The three Unsplash images use direct URLs with embedded crop parameters:

```
https://images.unsplash.com/photo-1460925895917-afdab827c52f?w=1200&h=700&fit=crop
https://images.unsplash.com/photo-1535378917042-10a22c95931a?w=800&h=600&fit=crop
https://images.unsplash.com/photo-1553877522-43269d4ea984?w=800&h=600&fit=crop
```

These are stock photos of generic dashboards and laptop screens—not product screenshots. The hero image locks the source at 1200×700 regardless of viewport; on mobile that's a 1200px download resized to ~375px display width.

The deeper risk: Unsplash's free tier has rate limits for hotlinking. A production site hitting these URLs on every visitor request will eventually get 429 responses or placeholder redirects. The standard mitigation is downloading images to `public/` or a project CDN during build.

## Dark-only design vs existing light-mode support

The old page used conditional dark-mode classes (`dark:bg-slate-950`, `dark:text-white`) throughout, inheriting the user's system preference or toggle state. The new page sets `bg-slate-950 text-slate-100` unconditionally on the root div. The nav removes all `dark:` prefixes—it's simply always dark.

This creates a visual discontinuity: if the rest of the app respects `prefers-color-scheme` or a theme toggle, users in light mode will see a dark landing page flash to a white dashboard on login. The fix is either forcing dark mode at the layout level (adding `className="dark"` to the html element for this route) or restoring conditional classes.

## Animation inventory: shipped vs used

Eight keyframe animations are added to `globals.css`:

| Animation | Used in page.tsx? |
|-----------|-------------------|
| `float` | Yes (floating shapes, sparkles) |
| `float-slow` | Yes (floating shapes) |
| `gradient-x` | Yes (hero headline) |
| `slide-in-bottom` | No |
| `pulse-ring` | Yes (integrations) |
| `orbit` | No |
| `typing-cursor` | Technically yes (the `|` blinks), but no typing effect exists—it's a static pipe character |
| `blob-morph` | No |

Three animations (`slide-in-bottom`, `orbit`, `blob-morph`) are dead code. The typing cursor gives the impression of a typewriter effect but no text is actually being typed—it's just a blinking pipe next to a static heading, which may confuse users expecting text to appear.

## Branding documentation as isolated artifact

`branding/BRAND_GUIDE.md` (374 lines), `branding/COPY_GUIDE.md` (318 lines), `branding/assets/colors.css` (192 lines), and `branding/assets/animations.css` (254 lines) are added but exist in isolation. The `colors.css` file defines custom properties like `--color-primary-600: #7c3aed` and the `animations.css` file defines keyframes like `@keyframes nucrm-float`—but neither is imported in `globals.css`, `layout.tsx`, or anywhere else. The landing page's `globals.css` additions define `@keyframes float` separately from `branding/assets/animations.css`'s `@keyframes nucrm-float`, creating parallel definitions of the same visual effect.

The copy guide suggests taglines like "Relationships, not spreadsheets" and "Enterprise CRM. Startup speed."—none of which appear on the redesigned page, which uses "Close deals faster with AI" instead.

## Removed functionality

The redesign removes `AnimatedCounter`—the only scroll-triggered interactive element on the old page. It also removes all testimonials and the "Starter" pricing tier. The `stats` array (500+ companies, 30% faster close rate, 99.9% uptime, 10K+ deals) is gone; the only social proof is a static "Trusted by 500+ teams" line with five placeholder company names rendered as text.
