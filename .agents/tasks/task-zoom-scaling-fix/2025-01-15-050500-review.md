# Percentage-based root font-size and rem sidebar widths for zoom scaling

The app's root `html` element had a fixed `font-size: 16px`, which blocks browser zoom from propagating through the rem unit chain. This change replaces both occurrences with `font-size: 100%` and converts user-preference overrides to percentage values (87.5%–125%). The two sidebar components (tenant and superadmin) had their widths and arbitrary px font-sizes converted from fixed pixels to rem equivalents so they scale with zoom.

Watch for: unconverted `text-[10px]` in other page components means zoom scaling is partial — sidebars scale but table headers and badges elsewhere do not (**confirmed**). The comment "overrides the base 16px set above" in globals.css is now stale (**confirmed**).

## High-level view

The root font-size fix is the load-bearing change. By switching from `16px` to `100%`, the browser's zoom multiplier now flows into every `rem`-based measurement. The user-preference overrides stack correctly because percentages on the root element multiply against the browser's effective base (including zoom).

The conversion was scoped only to the two sidebar components — elsewhere in the codebase, `text-[10px]` and similar arbitrary px values remain untouched, creating a partial migration where some UI elements scale and some don't.

## Root font-size: `16px` to `100%`

Two declarations set the html font-size — one inside `@layer base` (line 51) and one in the "Font rendering fix" section outside any layer (line 173). The duplicate is a pre-existing pattern where the second wins by source order. Both are updated.

The user-preference overrides become percentages that compound with zoom:

```css
html[data-font-size="small"]  { font-size: 87.5%; }   /* was 14px */
html[data-font-size="normal"] { font-size: 100%; }    /* was 16px */
html[data-font-size="large"]  { font-size: 112.5%; }  /* was 18px */
html[data-font-size="xl"]     { font-size: 125%; }    /* was 20px */
```

At 150% browser zoom with "large" preference, the effective root font-size becomes `16px * 1.5 * 1.125 = 27px`. Under the old px-based system, zoom would have been ignored entirely by these overrides.

## Incomplete migration outside sidebars

A grep for `text-\[.*px\]` across the codebase reveals unconverted px values in:

- `app/tenant/invoices/page.tsx` — `text-[10px]` on table headers
- `app/tenant/orders/page.tsx` — same pattern
- `app/tenant/quotes/page.tsx` — same pattern
- `components/tenant/report-builder.tsx` — `text-[10px]` on labels
- `app/tenant/settings/ai-providers/page.tsx` — `text-[10px]`, `text-[11px]`
- `app/auth/login/page.tsx` — `min-h-[44px]`, `w-[500px]`

At non-100% zoom, the sidebars will now scale while table headers and badges won't, creating a visual inconsistency between converted and unconverted components.
