VERSION STRATEGY
=====================================
- main (v1.0.0) = STABLE RELEASE — All 23 gaps resolved, production-ready
- feature/* branches = NEW FEATURES — Built in isolation, merged ONLY after confirmed working
- Each new version (v1.1.0, v1.2.0, etc.) = batch of confirmed features merged to main
- v1.0.0 is NEVER modified — it stays as the stable baseline

Branching Rules:
1. All new features start in feature/<name> branches
2. Feature branch must pass: tsc --noEmit, npm run test, npm run build
3. Feature is tested in isolated branch — if it breaks, delete branch and restart
4. Only merge to main after explicit confirmation from user
5. Merged features get a new version tag (v1.1.0, v1.2.0, etc.)
