# NuCRM Enterprise — Full Audit Report

**Date:** 2026-06-05  
**Total Issues:** 53 (4 critical, 20 high, 21 medium, 8 low)

---

## CRITICAL

- **1. JWT token leaked in login response body** — `lib/auth/api-handlers.ts:128-132` returns the JWT token in the JSON response body. Leaks to browser history, proxy logs, server logs, and browser extensions.

- **2. Real secrets committed to repo** — `.env.local` is tracked by git. Contains live `JWT_SECRET`, `SESSION_SECRET`, `ENCRYPTION_KEY`, `CRON_SECRET`, `SETUP_KEY`. Anyone with repo access can forge JWTs and decrypt data.

- **3. CSRF exemption for ALL `/api/auth/` routes** — `lib/auth/csrf.ts:145` exempts password reset, 2FA setup, email verification, invite acceptance from CSRF. Docs claim it was removed but code still has it.

- **4. CSRF "hash" is a fake 32-bit hash in Edge Runtime** — `lib/auth/csrf.ts:25-31` uses `String.hashCode()` (Java-style), not a real crypto hash. Edge runtime also can't use `require('crypto')`.

---

## HIGH

- **5. 50+ API routes leak `err.message` to production clients** — Raw error messages exposed in responses leak DB internals, file paths, and stack traces. Only half use `apiError()`.

- **6. `require()` used in ESM modules** — `lib/auth/csrf.ts:16,35` uses `require('crypto')` which throws `ReferenceError` in Edge Runtime / Next.js 16 strict ESM.

- **7. `.env.local` not in `.gitignore`** — Secrets permanently in git history with no way to remove.

- **8. `ignoreBuildErrors: true` in CI** — `next.config.mjs:15` ships 63 known TypeScript errors to production.

- **9. No `.dockerignore`** — Docker build context includes `.env.local`, `node_modules`, `.next`, secrets.

- **10. `DATABASE_SSL=false` in production configs** — `deploy/.env.production:77`, `docker-compose.yml:44`. DB traffic unencrypted.

- **11. Dockerfile hardcodes `JWT_SECRET="build-jwt-secret"`** — `Dockerfile:17`. Leaks into runtime image layers.

- **12. V2 API Gateway permits wildcard CORS by default** — `lib/api/gateway.ts:123`. Empty `allowedOrigins` = any origin allowed.

- **13. 103+ `.catch(() => {})` silent error swallows** — DB ops, webhooks, email, cron, backups all silently fail.

- **14. 190+ empty `catch {}` blocks** — Systemically hides all errors across the codebase.

- **15. `useEffect` without cleanup in 40+ components** — Memory leaks, state updates on unmounted components, duplicate requests.

- **16. Most API routes skip Zod validation** — Only ~30% use schema validation. Rest destructure raw `request.json()` unsafely.

- **17. `requireAuth()` wrapped in DB transaction** — `lib/auth/middleware.ts:99`. Every request opens a DB transaction. DB failure = false 401.

- **18. 200+ `as any` type assertions** — TypeScript safety defeated in critical paths.

- **19. Missing CSP + HSTS security headers** — `next.config.mjs:57-90`. No Content-Security-Policy or Strict-Transport-Security.

- **20. Grafana admin password defaults to "admin"** — `.env.example:85`.

---

## MEDIUM

- **21. No error.tsx boundaries on most superadmin pages** — Only templates have error boundaries. Crashes on dashboard, tenants, users pages blow up the whole layout.

- **22. Custom TOTP implementation instead of otplib/speakeasy** — `lib/auth/api-handlers.ts:83-89`. Custom base32 decoder has potential edge cases.

- **23. CSRF token endpoint rate limit too generous** — `ISSUES-2026-06-04.md:96`. 30 req/min allows spam.

- **24. SQL injection via dynamic table names** — `app/api/tenant/custom-fields/route.ts:65,183,191,225`. `${table}` interpolated directly into SQL query.

- **25. Proxy-based DB singleton loses type safety** — `drizzle/db.ts:15-19`. `as unknown` cast defeats type checking.

- **26. GDPR compliance has N+1 query pattern** — `lib/compliance/gdpr.ts:56-135`. 8 sequential queries instead of parallel.

- **27. SOC2 compliance same N+1 pattern** — `lib/compliance/soc2.ts:122-328`.

- **28. `requestToJson()` swallows JSON parse errors** — `app/api/forms/submit/route.ts:200-206`. Invalid JSON becomes `{}`.

- **29. Multiple `.json().catch(() => ({}))` patterns** — Several routes silently default invalid bodies to empty objects.

- **30. `verifySecret()` length check timing leak** — `lib/crypto.ts:17-19`. Early return on length mismatch leaks secret length.

- **31. BigInt serialization risk in metrics** — `app/api/metrics/route.ts:101`. `count(*)::int` may return BigInt on large datasets.

- **32. `dangerouslySetInnerHTML` in 5 components** — `lib/sanitize.ts:24` uses regex-based HTML stripping (bypassable).

- **33. Mixed ESM/CJS module patterns** — Various files mix `import` and `require`.

- **34. No log rotation for file logger** — `lib/logger.ts`. `nucrm.log` grows unbounded.

- **35. Grafana Loki errors silently dropped** — `lib/grafana.ts:320`. `flushLogs()` catch does `console.error` only.

- **36. No real-time alerting for errors** — No email/webhook/Slack notification for critical errors.

- **37. API error handler and logError don't coordinate** — `apiError()` sends to Sentry but doesn't persist to `error_logs` table.

- **38. `.catch(() => null)` in 6 places** — Returns silent null on failure.

- **39. `.catch(() => [])` in 17 places** — Returns empty array on failure, indistinguishable from "no results".

- **40. POST /api/superadmin/errors returns 500 with details** — `app/api/superadmin/errors/route.ts:112`. Production users see `details` field.

- **41. Cron jobs skip CSRF** — `lib/auth/csrf.ts:144`. Protected only by secret header.
